分析方法論

數據來源

  • HK 即時報價: Tencent qt.gtimg.cn (sub-1min delay，PE/PB/market_cap/52w 同時提供)
  • US 即時報價: YFinance (15-min delay)
  • 新聞: Futu Cloud (free tier, 60 req/min)
  • 歷史 bars: YFinance (HK 覆蓋不平均，新 foreign-listed 會 warn)
  • 1D forward returns: 由 compute_forward_returns.py 喺 user 朝早 run 時自動 fetch 上一個交易日 signal 嘅 next-day close（HK: Futu kline / US: yfinance batch）

分析流程

  1. Fetch 報價 + 技術指標 (MA20/50/100/200 + RSI + MACD + 成交量比)
  2. Fetch 新聞 (last 5 條 + sentiment)
  3. Prompt LLM 輸出 score_breakdown 嘅 4 個 dim (value/quality/momentum/order_flow) + 操作建議欄位 (entry/stop/target/support/resistance/sentiment/trend/operation_advice)
  4. Python 確定性加權：score = 0.05×value + 0.05×quality + 0.70×momentum + 0.20×order_flow（唔靠 LLM 自評分）
  5. Rule-based 決策引擎 override LLM 嘅 operation_advice（見下面）
  6. Logistic regression 計出 下日勝率（見下面）
  7. 寫入 SQLite + rebuild static HTML + push 到 Cloudflare Pages

4 維度評分模型 — 為 day-trade 度身訂造

維度權重睇咩
估值 value_score5%PE / PB / deviation from fair value
質素 quality_score5%ROE / margin / financial health / dividend stability
動能 momentum_score70%今日方向 / MA trend / RSI / deviation（純價格動能，唔包成交量）
資金流 order_flow_score20%量比 / 大單流入 / Relative Volume vs ADV / bid-ask imbalance / 北水

4 個 dim 嘅 score 0-100，只用嚟做「下日勝率」嘅 features，唔再獨立 display 畀用戶睇。Day-trade 完全偏重 momentum + order flow（合共 90%）。

Rule-Based 決策引擎（Phase 2，2026-07-10）

10-day audit 證明 LLM 直接嘅 operation_advice 係 ANTI-EDGE（LLM BUY 38.6% WR, LLM SELL 37.7% WR），所以我哋用 deterministic rules override LLM。Priority order：

  1. ANTI-CHASE: LLM 講 BUY + 樂觀 + m≥60 + chg≥+3% → 觀望（TOXIC pattern, 35.3% WR）
  2. ANTI-KNIFE: LLM 講 SELL + 悲觀 + chg≤-3% → 觀望（panic-day 會 mean-revert, 53/53 升）
  3. ANTI-MOMENTUM: LLM 講 BUY + m≥80 → 觀望（m 已 extended, 16.7% WR）
  4. CONSERVATIVE BUY: chg[-3,0) + sent非樂觀 + m[30,60] + 非科技 → 買入（61.5% WR, +0.92% avg, 23 records）
  5. BOUNCE BUY: chg[-5,-2] + sent悲觀/中性 + m<60 → 買入（51.7% WR, catches 7/2-style rebound, 445 records）
  6. DEFAULT: 其他 → 觀望（88% records, 38.6% WR if forced BUY）

LLM 嘅 operation_advice 會保存喺 llm_original_op column 供 audit，但 dashboard 只顯示 rule-based 決定。

📊 單一 score：下日勝率（Phase 4，2026-07-11）

解決「高 LLM 評分但 觀望」同「低 LLM 評分但 買入」嘅 confusion。方法：

  • 一個 score 解決所有事：每張 card 同 detail page 只顯示 「下日勝率 X%」，冇 LLM 評分、冇 訊號強度、冇 multi-hold、冇 ⚠️ 不宜追、冇 估值貴 etc.
  • 越高 = 越大機會 next day 贏：score 50 以上嘅信號 BUY 永遠高過 score 50 以下嘅 觀望
  • 校準自 1,822 個 1D forward returns（10 trading days，6/27-7/10）
  • 由 logistic regression 計出：features = m / of / v / q / chg / rule，out-of-sample 校驗：預測 30% bucket 實際 31.3% WR，預測 51% bucket 實際 54.7% WR

為何唔用 LLM 自評分？ LLM 訓練數據有 bullish bias — 信心分高往往 = LLM 寫得最 clean = post-rally 嘅時機，正好係散戶接棒嘅時間。10-day audit：LLM 80+ 信心分嘅 BUY 命中率得 16.7%。

下日勝率 → 操作 mapping：

  • 🟢 買入 永遠 >= 40%（CONSERVATIVE 78、BOUNCE 62）
  • 🟡 觀望 通常 14-40%（DEFAULT 38、ANTI-CHASE 24、ANTI-MOMENTUM 22）
  • 判斷：下日勝率 50 嘅 BUY 一定比下日勝率 50 嘅 觀望 強（已 lock 喺 rule 嗰度）

點解單一 score 重要？「高 = 大機會贏」係最直覺嘅決策軸。兩個 score 反而 split 注意力 — 用戶要同時記得 LLM 評分同 訊號強度嘅關係。1 個 score × 1 個 badge = 一眼就明。
