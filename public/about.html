關於我們 · AI 交易決策儀表板的由來與使命

最後更新: 2026-07-10 23:55 HKT

Leeks Terminal 係一個由散戶自建、自用、自負盈虧嘅 HK + US day-trade 決策儀表板。 我哋唔賣信號、唔做 subscription、唔做 PAMM。 我哋做嘅只有一件事：將自己用緊嘅 rule-based override 系統公開， 令同路嘅 retail traders 唔再俾 LLM 嘅「樂觀/悲觀」標籤呃到。

📌 Mission: Build the most honest, audit-first day-trade decision tool for HK+US retail traders. 全部決策邏輯公開、全部 backtest 結果公開、全部失敗案例公開。 贏就話贏，輸就話輸 — 唔做 cherry-pick。

點解我哋起呢個 dashboard？

2026 年 6 月，我哋用 MiniMax-M3 跑咗 10 個交易日嘅 live signal， 由 200 隻港股 + 200 隻美股出咗 1,913 個信號。 跟住做咗一個簡單 backtest：每個信號 lookahead 1 個交易日，計實際回報。

結果發現一件好 critical 嘅事：

LLM 反指實證 (n=1,913)

LLM 標記「樂觀」嘅股票，1 日 forward return 平均 -0.21%。

LLM 標記「悲觀」嘅股票，1 日 forward return 平均 +0.15%。

直接信 LLM BUY 信號嘅 hit rate：38.6%（仲差過 HOLD）。

換句話講，LLM 係反指標。佢訓練用嘅 corpus 大部分都係長線投資內容（trend-following 思維）， 但 day-trade 係 mean-reversion（超買回落、超賣反彈）。 兩個 regime 嘅方向完全相反 — LLM 一定係跟趨勢尾段入貨，即係所謂「chasing tops」。

呢個發現逼使我哋 重新設計整套架構：LLM 由「決策者」降級做「feature extractor」， 最終嘅 BUY / SELL / HOLD 決定權交返俾一組 deterministic rules。 重寫之後嘅 Conservative BUY hit rate 由 38.6% 升到 61.5%。

詳細數據、3 個 LLM trading mistakes、4 條 rule-based override， 全部寫咗喺 /insights.html（first-party audit 公開）。

我哋嘅 approach：Rule 先行，LLM 輔助

Leeks Terminal 嘅信號流程分兩層：

  1. Layer 1 — LLM feature extraction：MiniMax-M3 攞到即時報價、新聞、technical 指標之後， 輸出 4 維分數（Value / Quality / Momentum / Order-Flow）+ 結構化欄位（support / resistance / sentiment / trade_direction）。
  2. Layer 2 — Rule-based decision：Python 根據 deterministic rules（唔靠 LLM 自評）決定最終操作：
    • Anti-Chase：momentum ≥ 60 + sentiment 樂觀 + 今日漲 ≥ 3% → BLOCK BUY（chase top）
    • Anti-Knife：sentiment 悲觀 + 今日跌 ≥ 3% → BLOCK SELL（catch falling knife）
    • Conservative BUY：score 50-65 + 樂觀/中性 + chg < 0% + 過去 5 日 mean reversion 確認 → BUY
    • Bounce BUY：score 40-55 + 中性 + 過去 5 日跌 ≥ 5% + mean reversion 確認 → BUY（博反彈）

Rule 全部 deterministic，唔會因為 prompt 改變或者 LLM 換咗 model 而漂移。 每條 rule 都有對應嘅 backtest n / WR / avg return 喺 insights page 公開。
